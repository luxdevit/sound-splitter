# Contributing to Sound Splitter Studio

First off, thank you for considering contributing to Sound Splitter Studio! It's people like you that make this tool great.

## Where do I go from here?
If you've noticed a bug or have a feature request, make sure to check our Issues page to see if someone else has already created a ticket. If not, go ahead and submit a new one!

## Setting up your local environment
1. Fork the repository and create your branch from `main`.
2. Run `start_mac.command` or `start_windows.bat` to ensure the local environment establishes correctly.
3. Test your changes locally to ensure the python backend, UI and track separation still work perfectly.
4. If your change affects the User Interface, ensure the glassmorphism design language is respected.

## Submitting Pull Requests
- Keep your PRs focused on a single feature or bug fix.
- Ensure your PR description clearly describes the problem and solution.
- Remember to run basic tests to ensure you didn't break the native desktop container flow.
